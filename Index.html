Database Connectivity Hub

This system uses localStorage (Web Storage API) as a persistent, relational‑like database. All CRUD operations are handled entirely on the front‑end, demonstrating how a real back‑end would interact with a database.

DB Engine: Web Storage API (localStorage)
📀 DB Name: ExamSphereDB_v1
Tables: exams, students, results
Primary Keys: auto‑increment IDs
Relations: results.examId → exams.id, results.studentId → students.id (CASCADE delete)
// Front‑end Database Adapter (simulated RDBMS)
const db = {
  exams: () => JSON.parse(localStorage.getItem('exams') || '[]'),
  students: () => JSON.parse(localStorage.getItem('students') || '[]'),
  results: () => JSON.parse(localStorage.getItem('results') || '[]'),
  save: (key, data) => localStorage.setItem(key, JSON.stringify(data))
};
